# 截圖標註解讀協議

## 核心原則
**先讀懂所有標註，再動手。** 標註未全部理解前，禁止修改任何程式碼。

---

## 1. 標註符號對照表

| 符號 | 意義 | 行動 |
|------|------|------|
| ✕ / × / 叉叉（任意顏色） | **刪除** 該元素 | 移除對應的 JSX 節點或 CSS 規則 |
| 箭頭 ↑↓←→ | **移動** 該元素至箭頭指向位置 | 重新排列 JSX 順序或調整 CSS 定位 |
| 圓圈 ○ | **目標位置** 或「注意此處」 | 確認箭頭的目的地，或確認要調整的區域 |
| 數字編號 ①②③ | 操作**優先順序** | 依編號順序執行 |

---

## 2. 顏色代表「獨立操作組」

- 同一顏色的標註屬於同一組動作（例如：黃色箭頭 + 黃色叉叉 = 同一組改動）
- **不同顏色 = 不同元素的不同指示**，不可混淆

### 典型錯誤案例（已發生）
- 截圖中有**黃色叉叉**（刪除說明按鈕）和**藍色箭頭**（移動標題），被誤判為「兩個都是移動」
- 正確解讀：叉叉 = 刪除、箭頭 = 移動，顏色不影響符號意義

---

## 3. 解讀流程（強制執行）

```
步驟 1：列出截圖中所有標註
         → 每個標註：位置、顏色、符號、指向元素

步驟 2：對照符號表，逐一翻譯為「動作」
         → 例：「紅色 × 在說明按鈕上」= 刪除說明按鈕

步驟 3：確認每個動作是否有對應的程式碼位置
         → 若不確定元素對應哪個 JSX 節點，先 Read 檔案再判斷

步驟 4：若有任何標註模糊不清，立即詢問使用者，禁止猜測後直接動手

步驟 5：全部理解後，一次性執行所有修改
```

---

## 4. 收到「不對」時的強制停止規則

- 使用者說「不對」、「完全錯誤」、「錯了」時：**立即停止，禁止再次修改**
- 不得「再試一次」或「略做調整後繼續」
- 正確行動：
  1. 承認錯誤
  2. 重新閱讀使用者原始指示
  3. 用文字說明「我理解你的意思是 ___」，等待確認
  4. 確認後才執行

---

## 5. 常見標註語彙（此專案）

- 「照顏色移動」= 依各顏色標註的箭頭方向移動對應元素
- 「叉叉是刪除，其餘是移動」= 明確區分刪除與移動
- 「移到說明旁」= 移到 HelpCircle 按鈕的旁邊（通常是左側）
- 「移到頂列」= 移到 Topbar 顯示

---

## 6. 嚴禁行為

- ❌ 未讀完所有標註就開始改程式碼
- ❌ 將叉叉誤判為箭頭（或反之）
- ❌ 使用者說「不對」後繼續執行同方向的修改
- ❌ 猜測模糊標註的意思而不詢問
- ❌ 只處理部分標註，遺漏其他標註
