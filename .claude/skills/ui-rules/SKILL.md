# UI / 版面規範

以下規則適用於所有頁面的「實際視覺呈現效果」，不是單純參數設定。
在本次對話中的所有操作都必須嚴格遵守。

## 1. 對齊與視覺平衡
- 所有文字或區塊在實際畫面上「上下左右」都要視覺對齊
- 力求視覺平衡，避免參差不齊的排版

## 2. 消除無意義空白
- 沒有意義的空白間隙一定要縮到最小
- 可以列於同一列的元素就不要換行，除非資料特性需要分開顯示

## 3. 邊距要求
- 所有版面嚴禁貼著視窗邊線（必須保留適當邊距，貼邊很醜）

## 4. 內容完整性
- 新版本生成時，未經使用者允許**不得省略任何舊版本的內容**
- 這是最重要的規則，違反等同於刪除使用者的程式碼

## 5. 工作流程紀律
- 嘗試失敗 3 次後必須停止，擬定解決方案再執行排除問題
- 嚴禁重複作業超過 5 次，達上限時須告知使用者原因，經允許後才可繼續，否則停止

## 6. 字體規格違規的唯一合規解法
- 字體不得小於 10pt（≈13.33px），這是不可協商的底線
- 遇到「容器太小放不下合規字體」時，**唯一正確做法是擴大容器**
- **嚴禁以刪除內容、隱藏元素、建議移除文字來迴避規格問題**
- 典型錯誤案例（已發生）：進度條字體 10px 違規 → 誤建議刪除條內文字 → 正確解法是將進度條高度從 14px 加高至 22px，字體調至 13px
- 檢查順序：① 字體是否合規 → ② 若不合規，先調大字體 → ③ 若容器放不下，調大容器 → ④ 絕不刪內容
